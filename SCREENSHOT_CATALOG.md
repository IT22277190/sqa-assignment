## Compress / Convert / Crop Suites

These advanced suites rely primarily on Playwright failure artifacts (screenshot, trace, and video) configured in `playwright.config.ts`.

| Artifact Location | Description |
|---|---|
| `test-results/**` | Per-test execution artifacts, including retained trace/video on failure. |
| `playwright-report/` | Aggregated HTML report with test-level details and attachments. |

Covered test files:

- `compress_image.spec.ts`
- `convert_image.spec.ts`
- `crop_image.spec.ts`

Success screenshot folders:

- `screenshots/compress/` (e.g., `01_compress_load.png`, `02_compress_upload.png`)
- `screenshots/convert/` (e.g., `01_convert_jpg_load.png`, `02_convert_png_upload.png`)
- `screenshots/crop/` (e.g., `01_crop_jpg_load.png`, `02_crop_webp_upload.png`)

## Notes

- Screenshots are overwritten on each new run when the same file path is used.
- The current layout keeps Resize and Whole-System artifacts in separate folders so they do not collide.
- Advanced suites are configured with `screenshot: only-on-failure` and `video/trace: retain-on-failure` for cleaner success runs.
